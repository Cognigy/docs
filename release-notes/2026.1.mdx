## 2026.1.0

### Cognigy.AI

**Released** January 13, 2026

#### Features

##### Mocking

Introduced the [Mocking feature](/ai/agents/test/mocking). Mocking lets you change the behavior of a Node by replacing its logic with mock code execution without changing the original Node settings. This approach allows you to test specific Flow conditions in the Simulator, the Interaction Panel, and Endpoints without reconfiguring Nodes.

##### New Handover to Human Agent Tool

Introduced the [Handover to Human Agent tool](/ai/agents/develop/manage-ai-agents#handover-to-human-agent), allowing AI Agents to transfer conversations to human agents via handover providers if they can't address the user's request.

#### Improvements

- Added the Email Sender Name field to the [Email Notification Node](/ai/agents/develop/node-reference/data/email-notification)
- Published the [Confluence Extension 5.0.2](https://github.com/Cognigy/Extensions/releases/tag/confluence502) and [Diffbot Extension 1.0.2](https://github.com/Cognigy/Extensions/releases/tag/diffbot102) on the Cognigy Marketplace to address critical vulnerabilities of previous versions

#### Bug Fixes

- Fixed the issue where the max token limit in Cognigy.AI didn't allow custom LLM models to use the max token values set by the user
- Fixed the issue where Amazon Bedrock custom options didn't work properly
- Fixed the issue where MCP tools didn't support custom headers containing CognigyScript
- Fixed the issue where validation failed when saving AI Agent Nodes
- Fixed the issue where Markdown formatting wasn't rendered in messages in the Simulator
- Fixed the issue where the `.ack()` function was called in a different `amqp` channel than the one that received a message
- Fixed the issue where the goals list on the Create Scenario page didn't display the existing goals
- Fixed the issue where handover providers' icons weren't displayed in the Handover Provider list in the Check Agent Availability Node
- Fixed the issue where Packages including a period in the name missed the `.zip` extension in the file name
- Fixed the issue where HTTP requests in Endpoint transformers broke when the payload included `json: true`
- Fixed the issue where the Contact Profiles page in a Project displayed Contact Profiles from other Projects
