The table provides an overview of the LLM providers supported by Cognigy.AI, detailing the standard models and their compatibility with Cognigy.AI features.
You can also add a custom model that isn't listed in the table.
To do so, select the **Custom Model** parameter when creating a model and specify both the model type and name.
For more information about adding a custom or standard model for a selected LLM provider,
refer to the [All LLM Providers](/ai/agents/develop/gen-ai-and-llms/providers/all-providers) page.

  <sup id="footnote1">1</sup> [Design-time
  features](/ai/agents/develop/gen-ai-and-llms/generative-ai#design-time-generative-ai-features)
  include Intent Sentence Generation, Flow Generation, Adaptive Card Generation,
  Lexicon Generation.
</span>
<br />

<span style={{ fontSize: "0.8rem" }}>
  <sup id="footnote2">2</sup> Reasoning models consume more tokens and may incur
  higher costs. The models are optimized for tasks that require complex problem-solving
  and logical reasoning. Before using these models in production, test token consumption in [debug mode](/ai/agents/test/interaction-panel/chat#debug-mode) and use them with caution. To reduce costs, consider using a non-reasoning model such as `gpt-5-chat-latest`. For more information about reasoning models, refer to the [Microsoft Azure OpenAI](https://learn.microsoft.com/en-us/azure/ai-foundry/openai/how-to/reasoning), [OpenAI](https://platform.openai.com/docs/guides/reasoning/advice-on-prompting), and [Google](https://cloud.google.com/vertex-ai/generative-ai/docs/models) documentation.
</span>
<br />

<span style={{ fontSize: "0.8rem" }}>
  <sup id="footnote3">3</sup> For Knowledge AI, we recommend using
  `text-embedding-ada-002`. However, if you want to use `text-embedding-3-small`
  and `text-embedding-3-large`, make sure that you familiarize yourself with the
  restrictions of these models in [Which Model to
  Choose?](/ai/agents/develop/knowledge-ai/overview).
</span>
<br />

<span style={{ fontSize: "0.8rem" }}>
  <sup id="footnote4">4</sup> The `*-latest` suffix indicates that the model you
  select in Cognigy.AI points to the latest version of the model. For more
  information, read
  [Anthropic's](https://docs.anthropic.com/en/docs/about-claude/models/overview)
  or [Mistral
  AI's](https://docs.mistral.ai/getting-started/models/models_overview/) models
  documentation.
</span>
<br />

<span style={{ fontSize: "0.8rem" }}>
  <sup id="footnote5">5</sup> This feature is currently in Beta, hidden behind
  the `FEATURE_ENABLE_ALEPH_ALPHA_EMBEDDING_LLM_WHITELIST` feature flag, and may
  contain issues. Only one type of embedding LLM should be used per Project. If
  you choose to use `luminous-embedding-128`, you must create a new Project.
  Once you have chosen an embedding model for a Project, you cannot switch to a
  different embedding model; you must use a different Project. Failing to do so
  will result in errors while this feature is in beta.
</span>
<br />
